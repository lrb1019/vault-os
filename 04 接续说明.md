# Vault OS 接续说明

## 1. 当前实际架构

当前实现的核心不是概念设计稿，而是一个已经跑起来的 Obsidian 自定义视图：

- **入口文件**：`src/main.ts`
  - 注册自定义视图 `vault-os-view`
  - 提供 Ribbon 图标入口
  - 提供 `Open dashboard` 命令入口
  - 挂载设置页 `AgentDashboardSettingTab`
- **主视图文件**：`src/DashboardView.ts`
  - 当前实际渲染路径为：
    - 顶部状态栏 `renderTopTelemetry()`
    - 下方主频道区 `renderRightViewport()`
- **设置文件**：`src/settings.ts`
  - 负责看板标题、路径、MCP、TickTick、热力图尺寸、智能指令等配置
- **服务层**：
  - `src/services/VaultService.ts`
  - `src/services/DiaryService.ts`
  - `src/services/TaskService.ts`
  - `src/services/McpService.ts`
  - `src/services/ReadingService.ts`

## 2. 当前生效的界面结构

### 2.1 顶部状态栏
- **文件**：`src/DashboardView.ts` -> `renderTopTelemetry`
- **当前效果**：
  - 中间显示可配置标题 `dashboardTitle`
  - 右侧显示硬编码版本串和运行天数
  - 运行天数不再写死起始日期，而是改为按当前仓库中最早 Markdown 文件的真实创建时间计算

### 2.2 主频道区
- **文件**：`src/DashboardView.ts` -> `renderRightViewport`
- **当前实际生效的 5 个频道**：
  - `01 / 仓库`
  - `02 / 日记`
  - `03 / 巡检`
  - `04 / TickTick`
  - `05 / 项目`

### 2.3 未接入主渲染路径的代码
- `renderLeftControlBus()`、`renderNavigationBus()`、`renderClaudianWorkflows()`、`renderRecentFilesFeed()` 仍然存在，但当前 `render()` 没有挂载左侧栏。
- `ReadingService` 已实例化，但当前主界面没有实际渲染阅读数据。
- `renderMcpConsole()`、`renderTasksDashboard()` 等辅助渲染函数存在，但不是当前主频道入口的主路径。

这三点必须记住：代码里“存在”不等于界面里“已生效”。

## 3. 五大频道的当前实现

### 3.1 01 / 仓库
- **数据源**：`src/services/VaultService.ts`
- **当前实现**：
  - 单次扫描计算知识库文件分类统计和日期计数
  - 根据设置里的路径分类：
    - 日记
    - 收件箱
    - 项目
    - 原子笔记
    - 输出
    - 其他
  - 渲染文件占比条、迷你指标卡和图表区
  - 图表支持柱状图、日历视图和热力图切换
  - 月历视图与热力图虽然早期沿用了 `jarvis-stats-*` 的 class 风格，但当前命名体系已重构为插件自己的 `ad-stats-*`
  - 热力图单年/双年尺寸来自插件设置
- **性能机制**：
  - 使用实例级缓存 `cachedVaultOverviewStats`
  - 监听 `create/delete/modify` 事件后失效缓存
- **月历视图要点**：
  - 月历图并不是直接调用外部插件组件，而是本地 `renderCalendarChart()` 手动创建 DOM
  - 当前月历会先计算该月实际需要的周行数
  - 若月份落在 6 行网格中，会自动切换紧凑模式，压缩格子、间距和计数字号，以降低出现滚动条的概率

### 3.2 02 / 日记
- **数据源**：`src/services/DiaryService.ts`
- **当前实现**：
  - 支持 `day/week/month/quarter/year` 五种周期笔记
  - 优先读取 `notebook-navigator` 插件的命名规则和模板配置
  - 若可用，则尝试通过 `templater-obsidian` 创建笔记
  - 否则回退到插件内默认模板
  - 提供今日日记状态、周期统计、当前周期入口和“去年此时”摘要预览
- **注意**：
  - 这部分真实依赖外部插件配置，不是完全自洽的纯本地模块

### 3.3 03 / 巡检
- **数据源**：`src/services/VaultService.ts`
- **当前实现**：
  - 检查以下指标：
    - 收件箱积压
    - 待入库日记
    - 孤儿笔记
    - 死链
    - 空白笔记
  - 使用非线性扣分公式计算健康度
  - 点击条目可打开细项列表
  - 支持打开最近一次体检报告
  - 下方技能面板按钮来自设置中的 `claudianActions`
- **依赖边界**：
  - 巡检统计本身依赖 Obsidian metadata cache
  - 技能触发依赖 `realclaudian`

### 3.4 04 / TickTick
- **数据源**：`src/services/McpService.ts` 与 `src/services/TaskService.ts`
- **当前实现**：
  - 启动时优先读取本地缓存文件，保证界面先可用
  - 进入频道后触发 TickTick 同步
  - 通过插件设置中配置的 HTTP MCP 端点直接发起 JSON-RPC 请求
  - 当前覆盖的数据类型：
    - 项目列表
    - 未完成任务
    - 最近 30 天已完成任务
    - 习惯与打卡记录
    - 专注记录
  - 支持新增任务、完成任务、习惯打卡，并在本地缓存与远端之间做同步
  - 顶部提供同步状态徽标与小型刷新入口，不再使用“同步成功”的全局提示作为主要反馈
- **风险点**：
  - 强依赖插件内 TickTick 连接配置与远端 TickTick 服务可用性
  - 离线或未配置时会回退到 mock 提示数据

### 3.5 05 / 项目
- **数据源**：`projectBaseFilePath` 指向的 `Projects.base`
- **当前实现**：
  - 读取 `Projects.base` 内容并解析过滤条件
  - 遍历全库 Markdown，结合 frontmatter 和标签做匹配
  - 输出的是**表格视图**，不是 Kanban
  - 表格列当前包括：
    - 名称
    - 状态
    - 创建时间
    - topics
  - 顶部同时给出项目总数、活跃数、待办数、阻塞数、完成数

旧文档把这里写成 4 列看板，已经不符合实际。

## 4. 设置面板的当前口径

设置页当前分 4 个标签：

- `general`
  - 看板标题
  - 不再暴露热力图尺寸等低频技术参数
- `paths`
  - 日记路径
  - 收件箱路径
  - 原子笔记路径（巡检核心目录）
  - 项目目录
  - 项目 Base 文件路径
  - 归档目录
  - 输出目录
- `mcp`
  - TickTick 启用状态
  - TickTick 接口地址
  - TickTick 请求头
  - TickTick 服务名（高级项）
  - 同步防抖延迟
- `actions`
  - 可配置底部智能指令按钮
  - 支持 `{{input}}`
  - 支持 `{{daily_path}}`、`{{inbox_path}}`、`{{projects_path}}`、`{{archive_path}}`、`{{output_path}}`、`{{atomics_path}}` 等路径变量
  - 图标来源为 Lucide Icons

## 7. 双索引使用口径

本文件负责描述“当前真实代码状态”，但它本身不是代码真相源。后续接续时，必须配合两套索引一起使用：

- **管理索引**
  - 用来确认当前阶段目标、协作规则、需要同步更新的项目文档。
  - 主要配合 `[[00 项目总览]]`、`[[01 PROJECT_RULES]]`、`[[05 审查流程]]` 使用。
- **代码索引**
  - 用来确认某个频道、服务、渲染函数、命令入口当前是否真实接线。
  - 凡涉及“谁在调用它”“它又影响谁”“某段代码是否只是存在但未生效”，都应以代码索引核实，而不是只看本文件表述。

接续时的顺序固定为：

1. 先读 `[[00 项目总览]]`、`[[01 PROJECT_RULES]]`、`[[04 接续说明]]`。
2. 再用代码索引确认本次要改的入口函数、调用链和影响范围。
3. 改完后回到管理索引，决定是否更新 `[[03 改动日志]]`、本文件、`[[05 审查流程]]` 或 `[[07 审计与体检逻辑备忘录]]`。
