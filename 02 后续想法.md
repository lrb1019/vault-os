# 后续想法

- [x] 布局。关于布局，我的想法如下：整体分为上、左、右三个区域。
	- **上方区域：**显示 BYLRB 的个人主页。
	- **左侧区域（较窄）：约占1/4，长竖条的区域
		- 1. **快捷入口**：将一些常用文档做成快捷链接，点击即可打开，可自行设置。
		- 2. **快捷入口正下方**：使用 Navigator 插件展示最近打开的文件。
	- **右侧区域：约占3/4，长方形，分上边选项栏，下边展示区**上方为一排长条状选项，默认显示五个：笔记、日记、诊断、TikTok、项目等。点击任何一个就会显示器内部的内容，（默认显示笔记的）
		- **笔记**：默认显示整个仓库的相关数据，如笔记总数，以及笔记相关的热力图统计（显示在下方的较大区域）。
		-  **日记**：显示日记总数，以及一个统计图。例如，显示本月所有日记，有的显示绿色（已存在），没有的显示灰色，点击即可打开；若点击灰色区域，则基于 Navigator 的底层模板新建日记。
		- **诊断**：可考虑归入“笔记”中，用于显示需要消化的笔记数量、孤儿笔记数量以及仓库健康度。
		- **TikTok**：显示今日待办，可添加或勾选任务。另有一个同步选项，点击同步即可与 MCP 连接，抓取或上传数据。同时会显示习惯追踪、待办任务总数、本周完成情况等相关信息。
		-  **项目**：显示项目的状态分类，如不活跃、活跃、已完成、已放弃。后续可追踪对应笔记的属性。目前先保留框架，具体内容后续再填充。
